# Mobile UI Design Guide

## 언제 적용하는가

- 반응형 모바일 UI 구현 작업이 시작될 때
- 모달/다이얼로그에 모바일 대응이 필요할 때
- 페이지 레이아웃이 모바일에서 달라져야 할 때

## 사용자 확인 필수

아래 항목은 **반드시 사용자에게 먼저 확인**한다. 임의로 적용하지 않는다.

### 확인 항목

1. **모달 전환 방식**
   > "이 모달은 모바일에서 어떻게 처리할까요?"
   - 풀스크린 슬라이드
   - 바텀시트
   - 센터 모달 유지

2. **페이지 레이아웃 전환**
   > "모바일에서 레이아웃을 어떻게 바꿀까요?"
   - 사이드바 → 풀스크린 슬라이드
   - 그리드 → 세로 스택
   - 탭 → 스와이프
   - 그대로 유지 (스크롤만)

3. **버튼 하단 고정 여부**
   > "주요 액션 버튼을 모바일에서 화면 하단에 고정할까요?"

4. **키보드 대응**
   > "입력 폼이 있는데, 키보드 올라올 때 '완료' 바가 필요할까요?"

5. **레퍼런스**
   > "모바일 UI 레퍼런스가 있을까요? 없으면 참고할 서비스가 있나요?"
   - Toss (단계별 풀스크린 플로우, 하단 CTA 고정)
   - 당근마켓 (바텀시트, 리스트 중심)
   - 카카오톡 (탭 네비게이션, 채팅형 UI)
   - Airbnb (카드 스택, 풀스크린 필터)
   - 기존 프로젝트 내 다른 모바일 페이지

## 모바일 UI 패턴

사용자 확인 후, 아래 패턴을 프로젝트 컨텍스트에 맞게 적용한다.

### 1. 풀스크린 모달

데스크탑에서는 센터 모달, 모바일에서는 전체 화면으로 전환.

**핵심 규칙:**
- 하단에서 슬라이드인 트랜지션
- 키보드 올라오면 상단에 "완료" 바 표시
- 버튼 하단 고정 시 `flex flex-col` + `mt-auto` 패턴 사용
- Fragment(`<>`) 사용 금지 — flex chain이 끊김

### 2. 사이드바 → 풀스크린 전환

데스크탑은 사이드바+콘텐츠, 모바일은 메뉴 리스트 → 풀스크린 패널.

**핵심 규칙:**
- 모바일: `fixed inset-0 z-50` 풀스크린 패널 + 슬라이드 트랜지션
- 데스크탑: `hidden md:flex` 사이드바 레이아웃
- 뒤로가기 버튼 + 섹션 제목 헤더
- URL 파라미터 기반 섹션 관리 (브라우저 뒤로가기 지원)

### 3. 버튼 반응형

**금지:** 고정 width (`w-[184px]`) — 모바일에서 깨짐

**패턴:**
- 모바일 균등, 데스크탑 고정: `flex-1 sm:flex-none sm:w-[184px]`
- 모바일 세로, 데스크탑 가로: `flex flex-col-reverse gap-2 sm:flex-row sm:justify-end`

### 4. 고정 width 금지

모달/다이얼로그에서 `w-[Npx]` 사용 금지.

- `w-[442px]` → `sm:max-w-[442px]`
- 모바일은 기본 full width, 데스크탑만 max-w 제한

### 5. URL 파라미터 기반 네비게이션

모바일에서 브라우저 뒤로가기를 자연스럽게 지원하려면 `useState` 대신 URL 파라미터로 섹션 관리.

- 섹션 열기: `?section=profile`
- 뒤로가기: 파라미터 삭제
- `router.replace`로 히스토리 스택 관리

### 6. 키보드 대응

- 입력 폼이 있는 풀스크린 모달: 키보드 올라올 때 상단 "완료" 바
- 하단 고정 버튼: 키보드에 가려지지 않도록 처리
- iOS Safari: `visualViewport` API로 키보드 높이 감지

## 금지 사항

- 사용자 확인 없이 풀스크린 모달을 적용하지 않는다
- 사용자 확인 없이 하단 고정 버튼을 적용하지 않는다
- 고정 width (`w-[Npx]`)를 모바일 뷰에서 사용하지 않는다
- 프로젝트 기존 반응형 패턴을 무시하고 새 패턴을 만들지 않는다
